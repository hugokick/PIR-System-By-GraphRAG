# 展陈领域知识图谱构建方案

## 1. 目标

本方案用于在不干扰当前 MVP 主线的前提下，固定展陈领域知识图谱的实体和关系词表，并定义一套可测试、可离线运行的图谱构建流程。

本阶段只处理当前已有的结构化展项档案，不接入 OCR、文档切片、pgvector 或 LLM。

## 2. 固定实体

本阶段固定以下图谱实体：

- 展项 `exhibit`
- 项目 `project`
- 业主 `owner`
- 供应商 `supplier`
- 材料 `material`
- 主题 `theme`
- 交互方式 `interaction`
- 文档 `document`

## 3. 固定关系

本阶段固定以下图谱关系：

- 展项 -> 项目：`belongs_to_project`
- 展项 -> 业主：`owned_by`
- 展项 -> 供应商：`supplied_by`
- 展项 -> 材料：`uses_material`
- 展项 -> 主题：`has_theme`
- 展项 -> 交互方式：`has_interaction`
- 展项 -> 文档：`has_document`
- 展项 -> 相似展项：`similar_to`

## 4. 构建输入

当前构图输入来自现有后端领域对象 `ExhibitResponse` 列表，字段包含：

- 展项基础信息
- 项目、业主、供应商、主题
- 材料与交互方式列表
- 文档元数据
- 相似展项 ID 列表

## 5. 离线构图流程

离线构图流程如下：

1. 读取展项列表
2. 以每个展项为中心生成 `exhibit` 节点
3. 按结构化字段生成项目、业主、供应商、主题节点和关系
4. 按数组字段生成材料、交互方式和文档节点及关系
5. 按 `related_exhibit_ids` 生成相似展项关系
6. 输出统一图谱快照 `KGSnapshot`

图谱快照至少包含：

- `nodes`
- `edges`
- `evidences`
- `adjacency`
- `warnings`

## 6. 抽取规则

### 6.1 展项节点

每条展项记录生成一个 `exhibit` 节点，保留以下属性：

- 名称
- 类别
- 主题名称
- 场馆类型
- 状态
- 项目年份

### 6.2 单值实体

以下字段各生成一个实体节点，并与展项建立一条边：

- `project`
- `owner`
- `supplier`
- `theme`

### 6.3 多值实体

以下字段逐项生成节点和边：

- `materials`
- `interactions`
- `documents`

### 6.4 相似展项

`related_exhibit_ids` 映射为 `similar_to` 边。

如果目标展项在当前输入集中不存在：

- 不抛异常
- 不生成悬空边
- 在 `warnings` 中记录缺失目标

## 7. 证据模型

为支持后续 GraphRAG 引用来源，本阶段将证据与节点、边分离存储。

每条证据建议包含：

- `evidence_id`
- `source_type`
- `source_id`
- `title`
- `snippet`
- `linked_node_ids`
- `linked_edge_ids`

至少满足两类证据：

- 展项级证据：来自展项说明
- 文档级证据：来自文档标题和来源说明

## 8. 新增与编辑时的同步原则

本阶段先完成离线构图模块，不直接改现有 CRUD 流程，但固定后续接入原则：

- 新增展项时：生成以该展项为中心的节点、边和证据
- 编辑展项时：重建该展项中心子图，而不是局部打补丁
- 材料、交互方式、文档、相似展项关系采用“替换当前集合”的方式更新

这样可以避免多次编辑后产生关系漂移。

## 9. 与 MVP 主线的未来接口点

后续与主线对接时，优先通过以下接口点接入，而不是重写现有架构：

1. `repository.list_exhibits()` 输出 `ExhibitResponse` 列表给 KG 构建器
2. 现有图谱 API 可改为读取 `KGSnapshot` 输出节点和边
3. 后续 GraphRAG API 可直接复用同一份图谱快照

## 10. 本阶段不做

本阶段明确不做：

- 文件切片
- OCR
- Excel 导入构图
- pgvector 向量检索
- LLM 问答生成
- Neo4j 或图数据库迁移

## 11. 验收标准

本方案完成后，应满足：

- 图谱实体和关系词表固定
- 可以从现有展项结构化数据构建稳定图谱快照
- 缺失相似展项目标时不会报错
- 每个展项至少能产生一条可引用证据
- 构图模块可被单元测试覆盖
