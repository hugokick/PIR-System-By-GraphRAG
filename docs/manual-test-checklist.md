# 展项图鉴查询 MVP 手动验收清单

公网测试地址：

```text
http://106.52.200.183/pir-system/
```

建议每次里程碑部署后按本清单抽查。若公网无法访问，先确认服务器 Nginx、`docker compose -f docker-compose.cloud.yml ps` 和 `/pir-system/health` 状态。

## 1. 登录与角色

账号：

| 角色 | 用户名 | 密码 |
| --- | --- | --- |
| 管理员 | `admin` | `admin123` |
| 编辑 | `editor` | `editor123` |
| 访客 | `viewer` | `viewer123` |

验收点：

- 未登录时不能直接进入可写工作台
- 管理员可以看到审核操作、删除按钮、操作日志和日志导出
- 编辑可以新增、编辑、上传资料、导入表格、维护相似展项，但不能删除档案或资料
- 访客只能浏览、筛选、检索、查看图谱和资料
- 刷新页面后登录状态能恢复；退出后回到登录表单

## 2. 展项档案与结构化检索

验收点：

- 列表、详情和筛选数据正常加载
- 关键词可按名称、业主、供应商、标签检索
- 场馆、类别、主题、项目、业主、供应商、标签、材料、交互方式、状态、审核状态、预算区间筛选能影响列表结果
- 点击看板中的待审/退回指标能切换审核筛选
- 新增展项后能在列表和详情中看到，并可继续编辑
- 编辑已审核或已退回档案时，普通编辑保存后档案回到“待审核”

## 3. 资料与媒体档案

验收点：

- 图片、视频、PDF、Office、Excel/CSV、文本资料可上传
- 图片、视频、PDF 以紧凑缩略图或资料入口展示，并可打开预览弹窗
- Office、Excel/CSV 等资料可下载
- 文本类和可抽取文本的文档资料显示“已生成 N 个引用片段”
- 点击“字段建议”可看到抽取字段、置信度和来源片段
- 点击“套用到编辑表单”后，建议值会填入编辑表单，但不会自动保存
- 管理员删除误传资料后，资料列表刷新，操作日志出现删除记录
- 已审核或已落地档案的媒体/资料删除按钮不可用，并显示保护提示

## 4. Excel / CSV 导入

验收点：

- 下载模板可以获得 XLSX 文件，包含导入模板和字段说明
- CSV / XLSX 上传后先进入导入预览，不会立即写库
- UTF-8、UTF-8 BOM、GB18030 中文 CSV 可解析
- 中文模板表头和常见历史表头别名可映射，例如“档案编号、展品名称、所属项目、建设单位、项目 ID、预算 下限”
- 错误预算、缺少必填字段、重复展项编号会显示具体行和字段
- 相似展项字段引用不存在展项或自引用时会报错
- 同批导入的展项可以互相引用为相似展项
- 确认导入后，展项可在列表、详情和图谱中查看

## 5. 知识图谱

验收点：

- 当前展项模式调用 `GET /api/exhibits/{id}/graph`
- 全库演示模式调用 `GET /api/neo4j-demo/graph`
- 图谱标题为“Neo4j 知识图谱”，显示数据源、节点数、关系数
- 节点可拖拽，画布可缩放和平移
- 只有点击“重新布局”才重置布局
- 点击节点后显示节点详情，并高亮选中节点及其邻域
- 再次点击已选节点可取消选中
- 连续点击多个节点会累加高亮，不重排布局
- 边上可看到关系 type 或 label
- 相似展项关系新增或移除后，当前展项图谱会刷新

## 6. KG 关系推荐

验收点：

- 编辑或管理员登录后，详情页显示 KG 关系推荐区域
- 推荐只展示可采纳的 `similar_to` 候选
- 推荐项显示目标展项、置信度和推荐理由
- 点击“采纳”后，相似展项关系列表新增该展项
- 采纳关系会写入操作日志
- 非相似关系的镜像上下文不作为可采纳候选展示

## 7. 混合检索

建议问题：

```text
找几个适合低龄儿童、预算不高、互动性强的力学展项
类似杠杆乐园但预算更低
预算 20-30 万、儿童科技馆、机械互动
不要数字投影的水循环展项
```

验收点：

- 结果结合结构化筛选和自然语言意图
- 结果包含匹配原因，例如预算区间、主题、材料、互动方式、资料命中
- “预算不高 / 低预算 / 明确预算区间 / 类似某案例但预算更低 / 不要某条件”能影响排序或过滤
- 上传资料中的关键词命中后，匹配原因显示“匹配资料：资料名称”

## 8. GraphRAG 问答

建议问题：

```text
找几个适合低龄儿童、预算不高、互动性强的力学展项
根据上传资料说明一下热学互动台的设计依据
有没有类似杠杆乐园但预算更低的方案
definitely-not-in-library
```

验收点：

- 回答必须带编号引用，例如 `[1]`
- 来源卡片显示来源标题、`source_type`、snippet 和对应展项名称
- 文档来源卡可打开或下载原始资料
- 点击来源卡可定位到所属展项
- 答案区显示置信度和 warning
- 没有可引用来源时明确显示“未找到依据”或“未找到可引用来源”
- 有候选但缺少引用时，候选只作为参考，不进入正式可信答案
- 顶层 citations 只包含答案正文实际使用的来源

## 9. 管理与审计

验收点：

- 新增、编辑、审核、上传资料、删除资料、维护相似展项、导入表格都会写操作日志
- 操作日志动作显示中文标签，不暴露内部 action 名称
- 可按动作类型和资源编号筛选操作日志
- 管理员可导出当前筛选条件下的 CSV 日志
- CSV 使用中文表头和中文动作名
- 已审核或已落地档案不能直接删除；需先退回审核或变更状态
- 管理员调用 `GET /api/admin/system-status` 时可看到仓储类型、存储后端、认证模式、Neo4j 演示配置和展项/审计日志计数；编辑和访客访问应返回无权限

## 10. 成功标准总览

- 业务人员能录入真实展项
- 能按结构化条件快速筛选
- 能查看展项详情和资料
- 能看到展项与项目、业主、供应商、材料、主题、相似展项的图谱关系
- 能通过自然语言检索得到可解释结果
- 能基于库内资料进行初步问答，并返回引用来源
